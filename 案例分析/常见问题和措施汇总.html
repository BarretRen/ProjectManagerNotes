Project Manager Examine Notes

找案例中问题的技巧

  1. 技术人员出身担任项目经理：开发和管理是两条不同的主线，开发人员和管理人员所需要的技能是不同的。角色定位不好，没有站在项目经理的角度管理项目，需要培训
  2. 身兼数职：没有时间学习管理知识和从事管理工作，一人担任多个角色，工作负荷过重，给全局带来不利影响
  3. 使用新技术：技术风险，应该对大家进行培训、学习，监控技术风险，找合适的人从事新技术或选择外包
  4. 有人对项目不满意：没有建立有效沟通机制和方式，缺乏有效的项目绩效管理机制，加强沟通
  5. 变更相关：变更三要素**：书面申请、审批和确认、跟踪变更过程**
  6. 客户验收不通过：验收标准没有得到认可和确认，没有验收测试规范和方法
  7. 与人有关的问题或整治：沟通不到位，计划不够周详
  8. 过一段时间才发现问题：监控不力，不能及时发现问题
  9. 里程碑和仓促赶工：没有冗余考虑风险的想法
  10. 多头汇报：信息沟通不畅通、容易产生冲突，结合冲突产生的根源考虑

各领域常见问题

合同管理常见问题

  1. 没有签订合同，缺少某些条款
  2. 合同条款约定不明（时间、地点、质量要求、费用）
  3. 合同执行过程中没有相应的记录
  4. 项目变更没有相同的合同变更

变更中常见问题

  1. 没按照变更要求处理变更过程
  2. 变更要求没有留下书面记录
  3. 是否结束或拒绝变更，不能有项目经理独自决定
  4. 变更后没有相应的变更合同

项目管理计划的常见问题

  1. 计划制定应该全员参与
  2. 项目计划指定考虑不周，缺少对风险的考虑和应对措施
  3. 计划没有得到评审和批准就开始执行
  4. 项目情况发生变化后，没有相应的变更计划
  5. 计划的变更没有按照变更流程的要求进行

范围管理的常见问题

  1. 没有指定范围管理计划，或计划安排不合理
  2. 范围定义不充分，没做好需求分析和调研
  3. 缺少范围确认环节，需求和设计没有得到用户的正式评审
  4. 范围控制存在问题

进度管理的常见问题

  1. 没有指定进度计划，或计划安排不合理
  2. 没有预留应急时间储备或储备不足
  3. 活动资源估算不准确
  4. 项目存在延期问题时，没有及时更新进度计划，没有与XX沟通
  5. 压缩进度的方式不合理（赶工、快速跟进）
  6. 项目发生延期后没有按照变更流程处理延期

质量管理的常见问题

  1. 没有质量管理计划
  2. 质量职责分配不合理
  3. 没有QA，QA不独立与项目组，QA没有全程参与项目
  4. 质量控制缺少必要的环节，比如评审、测试
  5. 质量控制方法不合理、效果不好
  6. 没有按照变更流程处理质量的变更

人力资源管理的常见问题

  1. 组建团队问题，选人、存在问题
  2. 建设团队问题，没有有效的团队建设措施
  3. 没有对成员的绩效考核和激励机制
  4. 团队管理存在问题，没有及时发现冲突并分析问题，没有有效的冲突管理措施
  5. 没有做好沟通
  6. 经验不足
